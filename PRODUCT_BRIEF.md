# Product Brief

## Working title

Geo Run

## Concept

Geo Run is a mobile frontier settlement builder powered by real-world activity.
A player connects phone and wearable activity data, converts it into in-game
resources, and uses those resources to expand an outpost, unlock terrain, and
grow a living settlement.

## Core fantasy

The player is not closing rings or chasing step counts. They are building a new
home on a frontier. Daily activity powers construction, expansion, and gradual
transformation of a small outpost into a thriving settlement.

## Core loop

1. Real-world activity is imported from `Apple Health` or `Health Connect`.
2. Activity converts into construction and expansion resources.
3. The player returns to the settlement and invests those resources into
   buildings, terrain unlocks, and long-term upgrades.
4. Rewards feed visual growth, new modules, and new areas to develop.
5. The loop repeats without punishing skipped days.

## Activity mapping

- `Steps`: main universal resource for building and progression.
- `Floors`: secondary resource for difficult terrain, vertical builds, or
  special expansion paths.
- `Swimming` and other tracked workouts: optional rare resources for advanced
  terrain types, districts, or boosts.

The game must remain fun and complete using steps alone. Extra activity types
should expand options, not gate the main experience.

## MVP

- One frontier outpost.
- One hero or overseer as a light framing device, not the main gameplay system.
- Steps as the primary resource.
- Floors as a secondary bonus resource.
- A small set of buildings and expansions.
- Simple settlement progression.
- Gentle daily goals with no harsh penalties.

## Positioning

This is not a traditional fitness tracker and not a challenge app. It is a
mobile settlement-building game fueled by the player's real activity.

## Risks

- Rare activity types can create unfair unlocks.
- Settlement systems can grow scope too quickly.
- Purely passive building can become dull without meaningful choices.

## Guardrails

- Keep the city loop lightweight in the first version.
- Ensure the primary game loop works with steps alone.
- Avoid destructive setbacks in the MVP.
